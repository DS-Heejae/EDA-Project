1. 대회 및 프로젝트 소개

(1) kaggle 대회

  • 목적: 뉴욕에서의 택시 여행 기간을 예측하는 모델을 만드는 것
  • 성과 측정치(RMSE)가 가장 좋았던 사람을 뽑는 것보다는 통찰력 있고 사용 가능한 모델을 만드는 사람에게 보상을 지불하는 형태로 진행되었다.
    • pi : 예측값과 ai : 실제 값 의 차[cost function]를 제곱한 후 평균을 구해 루트를 씌어 주는 function
    • 0에 가까운 값이 나올수록 정확도가 높은 값이다.

(2) 프로젝트 목적

  • 본격적인 모델링에 앞서, EDA를 진행하며 train 데이터에 대한 이해를 높이고자 한다.

2. Data Preview

(1) 변수 설명

  • id: trip 별 고유 id
  • vendor_id: 택시 업체
  • pickup_datetime, dropoff_datetime: 승하차 일자(2016년 1~6월), 시간(시·분·초)
  • passenger_count: 택시 탑승객 수
  • pickup_latitude, pickup_longitude: 탑승 위·경도
  • dropoff_latitude, dropoff_longitude: 하차 위·경도
  • store_and_fwd_flag: 탑승객의 이전 탑승 기록 저장 여부
  • trip_duration : 이동 시간(초)

5. 가설 검증

  • 월별, 요일별 택시 pick up 횟수 확인 결과, 겨울(1-2월) 보다는 3-4월에 이용 빈도가 높다.
  • 시간대 별 택시 pick up 수가 18시~20시 사이에 가장 많다.

  • 시간대 별 택시 평균 이동 속도:
    • 점심시간(12~14시), 퇴근 시간(18~20)에 이동 속도가 가장 느린 것으로 보아, 이 시간 대에 교통 혼잡이 많이 일어남을 알 수 있다.

  • 승객수는 크게 영향을 주지 않는 것으로 보이며. 택시 탑승 위치는 뉴욕 맨하탄이 가장 많다.

→ 200개 샘플링 결과, 날씨는 크게 상관 관계를 발견하지 못하였다.

  • Darksky API에서 택시 탑승 시점의 날씨 데이터를 가져온 결과, 아래와 같이 분류되고 있다.
    • array(['Clear', 'Overcast', 'Possible Drizzle', 'Partly Cloudy', 'Mostly Cloudy', 'Light Rain', 'Drizzle', 'Possible Light Rain', 'Possible Flurries', 'Foggy'], dtype=object)
  • 날씨 변수와 이동 시간 변수 간의 상관 계수
    • 날씨 데이터는 One Hot Encoding 적용 하여 흐림(0), 맑음(1)으로 분류 ( 0 : Clear, Partly Cloudy // 1: Overcast, Lightsnow, Possible Drizzle, Light Rain)
    • 상관 계수: 0.02로 두 변수 간 선형 상관 관계가 매우 낮은 것으로 보인다.
  • 날씨에 따른 탑승 시각과 이동 시간의 분포: 플롯 상으로도 특별히 상관 관계가 없어 보인다.

  • 결과: 월별, 요일별 택시 pick up 횟수 확인 결과, 겨울(1-2월) 보다는 3-4월에 이용 빈도가 높음을 확인.
  • 결과: 택시 pick up 수가 18시~20시에 가장 많고, 동일 시간 대에 택시의 평균 속도가 가장 느림.
  • 결과: 승객수는 크게 영향을 주지 않고, 택시 탑승 위치는 뉴욕 맨하탄에서 가장 많은 것으로 확인.
  • 결과: 200개 샘플링 결과, 날씨는 크게 상관 관계를 발견하지 못하였다.
  • dark sky 하루 이용 건수(1000건) 제한으로, 날씨 데이터를 전부 다 가져오기에 한계가 있어서 200개로 샘플링하여 확인하였다. 샘플링에 따라 결과가 달라진다.

7. 프로젝트 회고

(1) Dark sky API 하루 이용 건수가 1000개로 제한되어 있어, 모든 날씨 데이터를 가져오는 데 한계가 있었다.

(2) 지도 상에 시간대 별 이동 경로를 표시하는 시각화 기법도 시도해보면 좋을 것 같다.

(3) 군집 분석 대신, 승객과 승하차 포인트를 세분화할 필요가 있을 것 같다.

(4) 공휴일과 주말을 구분해서 분석이 필요할 것으로 생각된다.

(5) 승객 수 별 tip duration 또한 심층 분석이 필요할 것 같다.
